# 天下風雲錄 Gameplay Staging Index

> 狀態：Staging Index
> 用途：作為 Gameplay 提案系統的入口、索引、分流與自動化路由。
> Canon Status：Non-Canon / Staging Only

---

## Purpose

本文件是 `07_Staging/Gameplay/` 的入口索引。

本文件不保存主要提案內容，也不作為正式 Canon。它只負責定義 Gameplay Staging 的檔案架構、寫入規則、彙整流程與版本轉入路徑。

---

## File Architecture

```text
07_Staging/Gameplay/
├── 天下風雲錄 提案.md          ← Index / Routing / Workflow
├── 天下風雲錄 提案資料.md      ← Raw Proposal Log
└── 天下風雲錄 提案彙整.md      ← Proposal Summary / PatchNotes Candidate
```

---

## File Roles

### 1. Index / Routing File

```text
07_Staging/Gameplay/天下風雲錄 提案.md
```

用途：

```text
1. 定義 Gameplay Staging 檔案架構。
2. 定義 AI 自動寫入與彙整流程。
3. 定義 Proposal Capture Automation。
4. 指向 Raw Proposal Log 與 Proposal Summary。
5. 不保存主要提案內容。
```

---

### 2. Raw Proposal Log

```text
07_Staging/Gameplay/天下風雲錄 提案資料.md
```

用途：

```text
1. 收錄使用者在遊戲中即時提出的 Gameplay 提案。
2. 保留使用者原始意圖與語境。
3. 保存提案提出時的章節、場景、測試想法與即時回饋。
4. 依 Proposal ID 追加紀錄。
5. 不負責最終採用與版本決策。
```

AI 行為：

```text
每當使用者在遊戲中提出新提案、測試結果、修訂意見或補充說明時，AI 必須優先寫入此檔案。
```

---

### 3. Proposal Summary

```text
07_Staging/Gameplay/天下風雲錄 提案彙整.md
```

用途：

```text
1. 將提案資料檔中的原始提案進行分類。
2. 合併相近提案。
3. 追蹤測試狀態。
4. 標記是否建議採用。
5. 撰寫可轉入 Official Patch Notes 的候選文字。
6. 作為下一版 Patch Notes 編寫前的整理來源。
```

AI 行為：

```text
每當提案資料新增、提案狀態改變、測試結果更新或使用者要求彙整時，AI 應同步檢查是否需要更新此檔案。
```

## Proposal Capture Automation

```text
PROPOSAL_CAPTURE_START

1. Detect user proposal or gameplay test feedback.
2. If the user introduces a new gameplay idea:
   - Generate or assign Proposal ID.
   - Write raw proposal into:
     07_Staging/Gameplay/天下風雲錄 提案資料.md
3. If the user revises or tests an existing proposal:
   - Update the matching Proposal ID in 提案資料.md.
4. If the proposal affects broader rules, UI, interaction structure, or future version design:
   - Update or create related summary in:
     07_Staging/Gameplay/天下風雲錄 提案彙整.md
5. Preserve original user intent.
6. Mark all entries as Non-Canon / Staging Only.
7. If a proposal becomes Accepted:
   - Prepare Patch Notes Candidate Text in 提案彙整.md.
   - Do not treat it as Canon.
8. Report updated file, Proposal ID, and current Status to the user.

PROPOSAL_CAPTURE_END
```

## Proposal ID Rules

```text
Format:
GP-YYYYMMDD-NNN

GP = Gameplay Proposal
YYYYMMDD = Created date
NNN = Daily serial number, starting from 001
```

---

## Status Rules

```text
Draft       初稿，尚未測試
Testing     遊戲中測試中
Accepted    測試通過，等待寫入 Official Patch Notes
Rejected    測試不通過，不採用
Revised     已修訂，需重新測試
Archived    保留紀錄，不再使用
```

---

## Canon Rules

```text
1. 本資料夾所有提案皆為 Staging。
2. Staging 不屬於正式 Canon。
3. Accepted Proposal 仍不等於正式規則。
4. Accepted Proposal 必須先整理為 Patch Notes Candidate Text。
5. 只有寫入 Official Patch Notes 後，才可成為正式 Canon 或正式 Gameplay Rule。
6. Staging 不得覆寫 Story Bible、Official Patch Notes、Runtime Save 或 Archive。
7. Staging 測試結果不得直接寫入 Runtime，除非使用者明確要求並仍須標記為測試紀錄。
```
