# Gameplay Proposal Capture Automation

## 目的

本自動化用於在遊戲進行中即時收錄、測試、追蹤與整理 Gameplay 提案。

當使用者於遊戲中提出任何玩法、互動、系統、選項、顯示格式、規則調整或測試想法時，AI 必須即時將該提案寫入本文件：

```text
07_Staging/Gameplay/天下風雲錄 提案.md
```

---

## 自動觸發條件

以下任一情況發生時，必須啟動 Proposal Capture Automation：

```text
1. 使用者提出新的 Gameplay 提案。
2. 使用者要求測試某項 Gameplay 規則。
3. 使用者修改、補充、否定或合併既有提案。
4. 遊戲實測中發現提案效果、問題或限制。
5. 使用者要求將測試結果保留至下次版本更新。
6. 使用者使用「提案」、「測試」、「加入暫存」、「下次更新」、「Gameplay」等明確語意。
```

---

## 寫入模式

```text
1. 新提案：Append 至「提案紀錄」區，不覆蓋舊內容。
2. 既有提案補充：更新同一 Proposal ID，不另建重複提案。
3. 測試結果：寫入該 Proposal ID 的 Test Result 欄位。
4. 狀態變更：更新 Status 欄位。
5. 正式採用：標記為 Accepted，但不得直接成為 Canon。
6. 正式移除：標記為 Rejected 或 Archived，不刪除紀錄。
```

---

## Proposal ID 規則

```text
格式：GP-YYYYMMDD-NNN

GP = Gameplay Proposal
YYYYMMDD = 建立日期
NNN = 當日流水號，從 001 開始
```

範例：

```text
GP-20260705-001
GP-20260705-002
```

---

## 提案狀態

```text
Draft       初稿，尚未測試
Testing     遊戲中測試中
Accepted    測試通過，等待寫入 Official Patch Notes
Rejected    測試不通過，不採用
Revised     已修訂，需重新測試
Archived    保留紀錄，不再使用
```

---

## 單筆提案固定格式

```text
## GP-YYYYMMDD-NNN：提案標題

Status: Draft / Testing / Accepted / Rejected / Revised / Archived
Created: YYYY-MM-DD
Updated: YYYY-MM-DD
Source: 遊戲中提案 / 測試回饋 / 系統整理 / 使用者指示
Scope: Gameplay / Interaction / UI / Narrative Flow / Rules / Other
Canon Status: Non-Canon / Staging Only

### Proposal

提案內容。

### Reason

提出原因、想解決的問題或預期效果。

### Test Plan

測試方式。

### Test Result

尚未測試 / 測試結果 / 問題 / 可行性。

### Decision

Pending / Accept / Reject / Revise / Archive

### Notes

補充說明。
```

---

## Canon 與版本更新規則

```text
1. 本文件所有內容皆為 Staging，不屬於正式 Canon。
2. Accepted 只代表測試通過，不代表已正式採用。
3. Accepted Proposal 必須整理進下一版 Official Patch Notes，例如：
   02_OfficialPatchNotes/天下風雲錄PatchNotes_vX.X.X.md
4. 只有寫入 Official Patch Notes 並納入版本更新後，才可成為正式規則。
5. 本文件不得覆寫 Story Bible、Official Patch Notes、Runtime Save 或 Archive。
6. 本文件永久保留提案演進紀錄。
```

---

## AI 執行指令

```text
PROPOSAL_CAPTURE_START

1. Detect whether the user is proposing, testing, revising, accepting, rejecting, or archiving a Gameplay idea.
2. If new proposal:
   - Generate next Proposal ID.
   - Append a new entry under 提案紀錄.
3. If existing proposal:
   - Locate matching Proposal ID or title.
   - Update the same entry.
4. Always preserve old proposal history unless the user explicitly requests cleanup.
5. Mark every entry as Non-Canon / Staging Only.
6. If a proposal passes testing, mark Status as Accepted and Decision as Accept.
7. Do not treat Accepted as Canon until it is written into Official Patch Notes.
8. After writing, report Proposal ID and updated Status to the user.

PROPOSAL_CAPTURE_END
```

# 提案一：隱藏互動選項（⑤）

## 說明

1. 遊戲預設互動選項：
   - ①～③：一般劇情選項
   - ④：自由行動
   - ⑤（隱藏）：特殊觸發選項

2. ⑤（隱藏）平時不顯示，僅於符合指定條件時出現。

3. 觸發條件包含但不限於：
   - 墨羽身分
   - 公開身分
   - 年齡
   - 能力
   - 聲望
   - 人物關係
   - 勢力關係
   - 劇情旗標
   - 世界事件
   - 真實歷史事件
   - 歷史戰役
   - 地點
   - 時間線
   - Story Bible 或官方資料庫定義之條件

4. 可觸發：
   - 特殊劇情
   - 隱藏事件
   - 歷史事件參與
   - 戰役參與
   - 特殊人物相遇
   - 特殊勢力互動
   - 特殊任務

5. 必須遵守：
   - Navigation.md
   - Story Bible
   - Official Character Files
   - World Database
   - Official Patch Notes
   - 不得違反官方 Canon。

6. 所有選項格式固定：
   - 每個選項皆必須包含「風險」。
   - 每個選項皆必須包含「備註」。
   - 「風險」與「備註」為固定欄位，不得因一般選項或隱藏選項而改變，也不得省略。

# 提案二：互動選項（①～③）人格／執念提示

## 說明

1. 劇情互動選項 ①～③ 於產生時，必須比對：

   MC-001_墨羽_Official_Character_File_v3.0.1

2. 若該選項符合墨羽 Official Character File 所定義之：
   - 真實人格
   - 執念

   則於「備註」中加入對應提示。

3. 提示內容應直接標示符合的人格或執念，例如：
   - 符合善良
   - 符合責任感
   - 符合初心
   - （後續依 Official Character File 擴充）

4. 風險與備註為固定欄位，不得省略。

5. 標示僅作為角色扮演提示，不代表最佳選項，亦不得影響劇情演算、公平性或玩家自由選擇。

---

## 顯示格式

① 主動扶起受傷的老人。  
風險：低｜備註：符合善良，可能影響人物關係與後續事件。

② 主動接下聚落分配的工作。  
風險：低｜備註：符合責任感，可能影響人物關係與後續事件。

③ 留下照顧孩童，讓其他大人前往警戒。  
風險：低｜備註：符合初心，可能影響人物關係與後續事件。
