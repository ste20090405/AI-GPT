# 天下風雲錄 Gameplay 提案入口

> 狀態：Staging Index
> 用途：定義 Gameplay 提案資料與提案彙整的分流規則。
> Canon Status：Non-Canon / Staging Only

---

## 檔案分工

Gameplay 提案系統自此拆分為兩個主要檔案：

```text
1. 07_Staging/Gameplay/天下風雲錄 提案資料.md
2. 07_Staging/Gameplay/天下風雲錄 提案彙整.md
```

---

## 1. 提案資料檔

```text
07_Staging/Gameplay/天下風雲錄 提案資料.md
```

用途：

```text
1. 收錄使用者在遊戲中即時提出的 Gameplay 提案。
2. 保存原始提案語意。
3. 保存測試想法、即時測試結果與補充回饋。
4. 依 Proposal ID 追加紀錄。
5. 不負責最終版本決策。
```

AI 行為：

```text
每當使用者在遊戲中提出新提案、測試結果、修訂意見或補充說明時，AI 必須優先寫入此檔案。
```

---

## 2. 提案彙整檔

```text
07_Staging/Gameplay/天下風雲錄 提案彙整.md
```

用途：

```text
1. 將提案資料檔中的原始提案進行分類。
2. 合併相近提案。
3. 追蹤測試狀態。
4. 標記是否建議採用。
5. 撰寫可轉入 Official Patch Notes 的候選文字。
```

AI 行為：

```text
每當提案資料新增或測試狀態改變時，AI 應同步檢查是否需要更新提案彙整檔。
```

## Proposal Capture Automation

```text
PROPOSAL_CAPTURE_START

1. Detect user proposal or gameplay test feedback.
2. Write raw proposal data into:
   07_Staging/Gameplay/天下風雲錄 提案資料.md
3. Preserve original user intent.
4. Assign or update Proposal ID.
5. If the proposal affects broader rules, update summary in:
   07_Staging/Gameplay/天下風雲錄 提案彙整.md
6. Mark all entries as Non-Canon / Staging Only.
7. If a proposal becomes Accepted, prepare Patch Notes candidate text but do not treat it as Canon.
8. Report updated file and Proposal ID to user.

PROPOSAL_CAPTURE_END
```

---

## Canon 規則

```text
1. 本資料夾所有提案皆為 Staging。
2. Staging 不屬於正式 Canon。
3. Accepted Proposal 仍不等於正式規則。
4. 只有寫入 Official Patch Notes 後，才可成為正式 Canon 或正式 Gameplay Rule。
5. Staging 不得覆寫 Story Bible、Official Patch Notes、Runtime Save 或 Archive。
```
