# Gameplay Rules Proposed v3.0.2

Status: Staging (Pending Review)

## 本文件用途
作為《天下風雲錄》遊戲規則的暫存區，待驗證完成後再整合至正式規則。

## 已確認規則

1. 沉浸式劇情文字敘述。
2. 全程第一人稱（墨羽視角）。
3. 每章目標長度 1500–2000 字。
4. 僅在單次回覆長度不足時，才拆分為「上／中／下」，拆分僅為輸出限制，不屬於正式章節制度。
5. 即使章節因輸出限制拆分為「上／中／下」，每一次遊戲回覆仍必須包含：劇情正文＋當前已知資訊＋目前未知資訊＋目前身分＋人物關係＋勢力關係（符合條件時）＋世界局勢（符合條件時）＋伏筆＋墨羽札記＋章節紀錄＋互動選項。
6. 劇情完全依據 Story Bible、MC-001_墨羽_Official_Character_File_v3.0.1 與 Official Patch Notes。
7. 每章開頭固定顯示世界時間。
8. 每次遊戲回覆固定顯示：當前已知資訊、目前未知資訊、目前身分（身分、所在地、公開身份）、人物關係、勢力關係（未加入可省略）、世界局勢（未加入勢力可省略）、伏筆、墨羽札記、章節紀錄。
9. 每次遊戲回覆提供互動選項，標示高／中／低風險，並保留自由行動。
10. 玩家知道＝墨羽知道，不使用上帝視角。
11. 世界持續運作，NPC、勢力與時間線不因玩家停止。
12. Canon 優先序：Story Bible → MC-001 → Official Character Files → World Database → GameData → Runtime。

## 13. 墨羽札記系統

### 13.1 功能定位

- 「墨羽札記」為墨羽親身記錄與回憶系統。
- 札記內容僅能來自墨羽親身經歷、親眼所見、親耳所聞、當下心境與合理推測。
- 不記錄墨羽未知資訊，不具上帝視角。
- 札記作為獨立資訊保存，方便日後劇情調閱與管理者模式檢索。

### 13.2 紀錄格式

```text
## 【墨羽札記】

【世界時間：隋・大業七年（611年）・初春・黎明】

（墨羽第一人稱札記內容）
```

- 每篇札記必須附完整世界時間。
- 札記以墨羽第一人稱書寫。
- 札記可記錄情緒、疑問、記憶、人物印象、地點印象與事件後感。

### 13.3 新增條件

- 墨羽札記不再限定夜晚才能新增。
- 只有在墨羽具備休息、停留、書寫或整理思緒的合理條件時，才能新增札記。
- 若墨羽正在逃亡、戰鬥、重傷、昏迷、被囚或其他不具備書寫條件的情況，則不新增札記。
- 若當前情境適合短暫整理思緒，可新增簡短札記；若情境安定，可新增完整札記。

### 13.4 劇情中自動調閱

- 當墨羽在劇情中遇到某件事、某個人、某個地點、某句話、某件物品或某種情境，若足以喚起過去記憶，系統可自動調閱相關札記。
- 劇情中調閱札記不限制兩天時間。
- 系統應依劇情需要調閱最相關的歷史札記，而非只顯示最近札記。
- 若沒有相關札記，則不顯示札記內容。
- 調閱札記必須符合「玩家知道＝墨羽知道」原則，不得藉札記透露墨羽未知資訊。

### 13.5 保存規則

- 墨羽札記作為 GameData 中的獨立資料保存。
- 札記不得覆蓋舊紀錄，應以時間序列累積。
- 管理者模式可完整調閱所有歷史札記。
- 遊戲模式中僅於劇情自然觸發時調閱相關札記。
