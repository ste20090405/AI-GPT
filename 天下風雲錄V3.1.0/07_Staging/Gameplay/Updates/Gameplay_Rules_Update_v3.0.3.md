## 一、更新目的

本更新檔依據 `Gameplay_Rules_Proposed_v3.0.3.md` 製作，用於整理 v3.0.3 提案中的規則變更重點，作為後續整合至正式 Gameplay Rules 或 Official Patch Notes 的過渡文件。

本檔案不直接取代正式 Canon；若與 Story Bible、Official Character Files、GameData 或正式 Patch Notes 衝突，仍以正式文件為準。

---

## 二、主要更新項目

### 2.1 核心敘事模式確認

- 遊戲敘事維持沉浸式文字敘述。
- 全程採墨羽第一人稱視角。
- 嚴格遵守「玩家知道＝墨羽知道」。
- 不使用上帝視角。
- 不透露墨羽尚未得知的幕後真相、他人內心、未來事件或隱藏情報。

---

### 2.2 章節長度與拆分規則

- 每章目標長度為 1500–2000 字。
- 「上／中／下」僅作為單次回覆長度限制下的拆分方式。
- 「上／中／下」不屬於正式章節制度。
- 即使拆分，每次回覆仍須包含完整狀態資訊與互動選項。

---

### 2.3 每次遊戲回覆固定結構

每次遊戲回覆應包含：

1. 劇情正文
2. 【當前已知資訊】
3. 【目前未知資訊】
4. 【目前身分】
5. 【人物關係】
6. 【勢力關係】（符合條件時顯示）
7. 【世界局勢】（符合條件時顯示）
8. 【伏筆】
9. 【墨羽札記】
10. 【章節紀錄】
11. 【互動選項】

---

### 2.4 世界時間開頭格式

- 每章或每次劇情回覆開頭必須加入世界時間。
- 世界時間須依 Story Bible、World Database、GameData 與劇情推進合理變化。

建議格式：

```text
**世界時間：**
**隋・大業七年（611年）・初春・黎明**
```

---

### 2.5 動態資訊系統

以下資訊改為動態顯示：

- 【當前已知資訊】
- 【目前未知資訊】
- 【人物關係】

規則重點：

- 不固定列出所有設定。
- 只顯示目前仍重要、仍影響墨羽判斷或劇情推進的資訊。
- 已解決、失效、過期或不再重要的資訊應移除或轉換。
- 不顯示墨羽尚未察覺的秘密。
- 人物關係不無限制累積，長時間退場或暫不影響劇情的人物可暫時不顯示。

---

### 2.6 章末資訊固定順序

每次遊戲回覆章末資訊採以下順序：

```text
## 【當前已知資訊】

## 【目前未知資訊】

## 【目前身分】

## 【人物關係】

## 【勢力關係】

## 【世界局勢】

## 【伏筆】

## 【墨羽札記】

## 【章節紀錄】

## 【互動選項】
```

備註：

- 【勢力關係】若墨羽未加入任何勢力，或目前無可顯示勢力關係，可省略。
- 【世界局勢】僅能顯示墨羽透過合理管道取得的資訊；無可靠情報時可省略。

---

### 2.7 互動選項規則

- 每次遊戲回覆固定提供四個互動選項。
- 前三項由系統依據當前劇情、墨羽能力、身份、處境、已知情報與世界狀態動態產生。
- 第四項固定為「自由行動」。
- 每個選項皆須標示：
  - 風險：高／中／低（或依玩家行動判定）
  - 影響：說明可能造成的影響。
- 不提供墨羽目前不可能想到或無法執行的選項。

標準格式：

```text
① ……

風險：中

影響：……

② ……

風險：低

影響：……

③ ……

風險：高

影響：……

④ 自由行動

風險：依玩家行動判定

影響：由世界狀態、角色能力與玩家選擇共同決定。
```

---

### 2.8 資訊同步原則

以下區塊必須即時同步更新：

- 【當前已知資訊】
- 【目前未知資訊】
- 【人物關係】
- 【勢力關係】
- 【世界局勢】
- 【伏筆】

不得無限制累積已失效、已解決、已過期或不再重要的資訊。

---

### 2.9 目前身分規則

【目前身分】只顯示：

- 身分
- 所在地
- 公開身份

不顯示姓名，因玩家固定以墨羽第一人稱進行遊戲。

---

### 2.10 墨羽札記系統

墨羽札記定位：

- 作為墨羽親身記錄與回憶系統。
- 內容僅能來自墨羽親身經歷、親眼所見、親耳所聞、當下心境與合理推測。
- 不記錄墨羽未知資訊，不具上帝視角。
- 作為獨立資訊保存，方便日後劇情調閱與管理者模式檢索。

新增條件：

- 不限定夜晚才能新增。
- 只有墨羽具備休息、停留、書寫或整理思緒的合理條件時，才能新增札記。
- 逃亡、戰鬥、重傷、昏迷、被囚等狀態下，不新增札記。

劇情中自動調閱：

- 遇到足以喚起記憶的人、事、地、物、語句或情境時，可自動調閱相關札記。
- 不限制兩天時間。
- 應調閱最相關歷史札記，而非只顯示最近札記。
- 必須符合「玩家知道＝墨羽知道」原則。

保存規則：

- 作為 GameData 中的獨立資料保存。
- 不得覆蓋舊紀錄，應以時間序列累積。
- 管理者模式可完整調閱所有歷史札記。
- 遊戲模式中僅於劇情自然觸發時調閱相關札記。

---

### 2.11 世界運作原則

- 世界持續運作，NPC、勢力與時間線不因玩家停止。
- 玩家選擇可影響人物命運、人際關係、地方局勢、支線事件、江湖聲望與自身人生。
- 玩家不得推翻 Story Bible 已鎖定的歷史節點。
- 所有自由皆建立於世界法則、歷史時間線、人物能力、身份限制、社會制度與生命規則之內。

---

### 2.12 Canon 優先序

Gameplay 執行時依以下順序判定：

1. Story Bible
2. MC-001_墨羽_Official_Character_File_v3.0.1
3. Official Character Files
4. World Database
5. GameData
6. Runtime Game Data

若暫存規則與正式 Canon 衝突，以正式 Canon 為準；若已發布 Official Patch Notes 修訂，依 Patch Notes 指示處理。

---

### 2.13 模式切換規則

#### 遊戲模式（預設）

- 不需輸入 `@GitHub`。
- 所有輸入皆視為墨羽的行動、對話或互動選項。
- 劇情依最新 Runtime GameData 持續推進。
- 回覆格式必須遵循 Gameplay Rules 規範。

#### GitHub 管理模式

- 使用 `@GitHub` 作為指令開頭。
- 所有輸入皆視為 Repository 管理操作，不推進遊戲劇情。
- 可讀取、建立、修改、同步：
  - Story Bible
  - Official Character Files
  - World Database
  - Gameplay Rules
  - Patch Notes
  - GameData
  - Runtime 存檔

#### 模式切換

- 使用 `@GitHub` 即進入 GitHub 管理模式。
- 未使用 `@GitHub` 時，預設為遊戲模式。
- 離開 GitHub 管理模式後，遊戲將依最新 Runtime GameData 無縫接續，不需重新載入。
