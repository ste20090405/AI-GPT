# 07_Documents/Git

本目錄保存 Git、Commit、Changelog 與 Release 相關規範。

原 `08_Changelog/README.md` 已依 Appendix E Repository Guidelines 整合至本目錄與根目錄 `CHANGELOG.md`。

---

## 職責

- Git Commit Convention。
- Release Tag 規範。
- Branch 建議。
- Changelog 規格。
- 記錄資料庫結構調整。
- 記錄重大資料移動。
- 記錄 Canon 整合摘要。
- 輔助對照 Git Commit。

---

## Changelog 與 Patch Notes 的差異

| 類型 | 用途 |
|---|---|
| `05_PatchNotes` | 對外正式版本更新說明 |
| `CHANGELOG.md` | Repository 層級維護與結構變更摘要 |
| `07_Documents/Git` | Git、Commit、Release 與 Changelog 規範 |

---

## 原則

1. Commit Message 應遵循 Appendix H。
2. Release 與 Version 應遵循 Appendix C。
3. Git History 應支援長期追溯。
4. Changelog 不取代 Patch Notes。
5. Changelog 不保存正式設定正文。
6. 每次重大資料庫重構後，應補充摘要。
7. 舊紀錄保留，不覆蓋。

---

## 維護狀態

- Status: Planned / Active
- Canon Level: Maintenance Record
